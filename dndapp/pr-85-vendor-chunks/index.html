Vendor Chunk Split

PR #87 · Closes #85 · Build fix

TL;DR: The Cloudflare production build has been failing on every merge to main for weeks — not a recent regression. Root cause is vite-plugin-pwa's 2 MiB workbox precache limit: the main JS bundle was teetering right at the cliff (2.0 MB local, 2.2 MB on Cloudflare's bun-based install). Fix is 18 lines in vite.config.ts that split phaser / react / dexie into their own chunks. Main bundle drops 67% (2,015 → 653 KB) and every chunk lands comfortably under the limit.

The real error

Pulled from the Cloudflare Workers build dashboard. The issue filed (#85) guessed at likely suspects (env vars, Node version, bundle-size-to-Workers-limits). All wrong — the error had nothing to do with Cloudflare. It's a local vite-plugin-pwa / workbox constraint that just happened to manifest in Cloudflare's build environment first.
