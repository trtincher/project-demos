PR #64 — Resilience, Chat & Table Lifecycle (Phase 9C)

Feature branch: feat/phase-9c-resilience-polish | Closes #58–#63

Overview

The final multiplayer polish phase adds disconnect handling with AI takeover, reconnect recovery, in-game chat, and full table lifecycle management (return to lobby, disband table).

1. Disconnect Detection

When a player loses connection mid-combat, presence leave events trigger handlePlayerDisconnect on the host. The player is marked as disconnected in the database.

📸 Screenshot: Player disconnect notification in combat

2. AI Takeover (60s Timeout)

After 60 seconds of disconnect, the host runs AI (decideAction) for the disconnected player's character — one move + attack, then auto-ends turn. Chains automatically if multiple players are disconnected.

📸 Screenshot: AI taking over a disconnected player's turn

3. Reconnect Recovery

When a player reconnects, the host re-sends combat_start (with ownership map) and state_sync so the player immediately sees the current combat state and can resume control.

📸 Screenshot: Player reconnecting and seeing current combat state

4. In-Game Chat

A collapsible ChatPanel component with message history (last 100 messages), text input, and send button. Uses chat_message broadcast on the Realtime channel. Available in both lobby and combat.

📸 Screenshot: Chat panel open with messages between players

5. Return to Lobby

After combat ends (victory or defeat), the DM can click "Return to Lobby" to reset combat state and bring all clients back to the lobby for another encounter. No need to create a new table.

📸 Screenshot: Return to Lobby button after combat ends

6. Disband Table

The DM can disband the table entirely, which closes the session and disconnects all clients. Used when the game session is over.

📸 Screenshot: Disband Table confirmation

Technical Highlights

  • Presence-based disconnect — Realtime presence leave triggers host-side handler
  • AI takeover — Reuses existing decideAction AI for disconnected player turns
  • Reconnect sync — Full state re-send on presence join during combat
  • Chat system — Broadcast-based with in-memory store and listener pattern
  • Lifecycle events — CombatEvent and LifecycleEvent type unions in multiplayer.ts
