PR #48 — Multiplayer Foundation (Phase 9A)

Feature branch: feat/phase-9a-multiplayer-foundation | Closes #41–#47

Overview

The multiplayer foundation adds real-time lobby infrastructure powered by Supabase Realtime. Players can host or join tables using a 6-character code, select characters, pick encounters, and launch into combat together.

1. Multiplayer Entry — Host or Join

New Multiplayer page accessible from the roster. The host creates a table and gets a shareable code. Other players enter the code to join.

📸 Screenshot: Multiplayer page with Host Table / Join Table options

2. Live Lobby — Real-Time Player List

Once in a table, all players see a live lobby with presence tracking. Each player selects a character from their roster. The host picks an encounter and sees everyone's ready state in real time.

📸 Screenshot: Lobby with player list, character selection, ready states

3. Character Selection

Players choose from their saved roster. Selection updates broadcast instantly to all lobby participants via Supabase Realtime presence.

📸 Screenshot: Character picker modal in lobby

4. Encounter Picker (Host Only)

The host selects from prebuilt scenarios or custom encounters. All players see the selected encounter details update live.

📸 Screenshot: Encounter picker modal showing prebuilt + custom encounters

5. Connection Status

A persistent connection indicator shows Realtime channel status. Players see when they're connected, reconnecting, or disconnected.

📸 Screenshot: Connection status indicator

Technical Highlights

  • Supabase migration for combat_sessions and session_participants tables with full RLS policies
  • Realtime channel management library (multiplayer.ts) for broadcast + presence
  • 7 new session CRUD functions in cloudDb.ts
  • multiplayerStore — Zustand store managing full lobby lifecycle
  • Routes (/multiplayer, /table/:code) and nav integration
