Problem

Once a campaign was created, its elements were completely static. DMs had to delete and recreate entire campaigns just to adjust party members, reorder encounters, or rebalance difficulty mid-session.

Before / After

Before

  • Campaign name & description: read-only
  • Party members: frozen at creation
  • Encounters: fixed order, no add/remove
  • Enemy composition: unchangeable
  • Party per encounter: always full party

After

  • Click-to-edit name & description
  • Add/remove party members anytime
  • Add/remove/reorder encounters
  • Scale enemy counts & types per encounter
  • Select which party members per encounter

Capabilities

R1 Metadata Editing

Click campaign name or description to edit inline. Save on Enter/blur, cancel on Escape. Empty names rejected.

R2 Party Management

Add characters from roster with toggle picker. Remove with confirmation dialog. Removing a member auto-cleans per-encounter overrides.

R4 Encounter Management

Add encounters from library via modal with search. Remove pending encounters. Reorder with Up/Dn controls. currentEncounterIndex auto-recalculates.

R3 Per-Encounter Party

Toggle which party members participate in each encounter. "Use Full Party" resets. Battle launch respects the override. 3/5 party badge shown.

R5 Encounter Scaling

Adjust enemy count with +/- controls (max 10). Add/remove enemy types from monster library. Live difficulty badge recalculates. Customized badge shown. "Reset to Original" clears overrides. Original encounter template never modified.
